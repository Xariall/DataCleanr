# API Reference — DataCleanr

Base URL: `https://your-app.railway.app` (update after deploy)
Interactive docs: `/docs` (Swagger UI, auto-generated by FastAPI)

---

## Authentication

All protected endpoints require:
```
X-API-Key: dc_<your_key>
```

Get a key: `POST /register`

---

## POST /register

Register a free-tier account and receive an API key.

**Auth:** None required

**Request:**
```http
POST /register
Content-Type: application/json

{"email": "you@example.com"}
```

**Response 200:**
```json
{
  "api_key": "dc_aBcDeFgH...",
  "message": "Store this key safely — it will not be shown again."
}
```

**Errors:**
| Code | HTTP | Meaning |
|------|------|---------|
| `INVALID_EMAIL` | 400 | Missing `@` or invalid domain part |
| `EMAIL_EXISTS` | 409 | Email already registered |

## POST /transform

Transform a file using plain English instructions.

**Auth:** Required
**Quota:** Deducts rows from daily budget (500 free / 500K paid)

**Request:**
```http
POST /transform
X-API-Key: dc_...
Content-Type: multipart/form-data

file=@data.csv          # CSV, JSON, or xlsx — max 10 MB
instructions=remove rows where email is empty and standardize dates to ISO 8601
```

**cURL example:**
```bash
curl -X POST https://your-app.railway.app/transform \
  -H "X-API-Key: dc_..." \
  -F "file=@data.csv" \
  -F "instructions=remove rows where email column is empty"
```

**Response 200:**
```
Content-Type: text/csv
X-DataCleanr-Summary: Removed 2 rows, 1 rows remaining
X-DataCleanr-Warning: Removed 67% of input rows - verify instructions  (if >30%)

name,email,score
Alice,alice@example.com,90
```

**Response headers:**

| Header | Always | Meaning |
|--------|--------|---------|
| `X-DataCleanr-Summary` | Yes | Rows in/out summary |
| `X-DataCleanr-Stats` | Yes | JSON dict with full stats |
| `X-DataCleanr-Warning` | If >30% rows removed | Data loss warning |

**Errors:**
| Code | HTTP | Meaning |
|------|------|---------|
| `MISSING_API_KEY` | 401 | No `X-API-Key` header |
| `INVALID_API_KEY` | 401 | Key not found |
| `EMPTY_INSTRUCTIONS` | 400 | Blank instructions field |
| `INSTRUCTIONS_TOO_LONG` | 400 | > 2000 characters |
| `FILE_TOO_LARGE` | 413 | > 10 MB |
| `INVALID_FILE` | 400 | Unparseable file |
| `EMPTY_FILE` | 400 | File has no data rows |
| `RATE_LIMIT_EXCEEDED` | 429 | Daily row quota exhausted |
| `BLOCKED_INSTRUCTIONS` | 400 | LLM output triggered AST deny-list |
| `UNINTERPRETABLE_INSTRUCTIONS` | 400 | Claude returned noop marker |
| `EMPTY_RESULT` | 400 | Transform removed all rows |
| `LLM_UNAVAILABLE` | 502 | Anthropic API error after 3 retries |
| `TRANSFORM_TIMEOUT` | 502 | Subprocess exceeded 30s |
| `TRANSFORM_FAILED` | 502 | Subprocess exited non-zero |
| `SERVICE_UNAVAILABLE` | 503 | Redis down (free tier only) |

All error responses include a `"try"` field with a working cURL example.

## POST /preview

Same as `/transform` but processes only the first 10 rows. No quota deduction.

**Auth:** Required (no quota deduction)
**Timeout:** 3 seconds (vs 30s for /transform)
**Use case:** Validate instructions before running on full file

**Request:**
```http
POST /preview
X-API-Key: dc_...
Content-Type: multipart/form-data

file=@data.csv
instructions=remove rows where email is empty
```

**Response 200:**
```
Content-Type: text/csv

name,email
Alice,alice@example.com
```
(maximum 10 data rows)

---

## POST /explain

Describe what a transformation will and won't do — without uploading a file.

**Auth:** Required
**Quota:** Costs 1 row against daily budget (abuse prevention)

**Request:**
```http
POST /explain
X-API-Key: dc_...
Content-Type: multipart/form-data

instructions=remove rows where email is empty and standardize all dates to ISO 8601
```

**Response 200:**
```json
{
  "will": "Remove rows where the email column is null or empty, and convert date columns to ISO 8601 format (YYYY-MM-DD).",
  "will_not": "Modify rows with valid emails or affect non-date columns."
}
```

## POST /webhook/stripe

Internal Stripe webhook. Not for direct use.

Stripe events handled:
- `checkout.session.completed` → upgrade user to PAID
- `invoice.payment_failed` → set `payment_failing` flag (no downgrade)
- `invoice.payment_succeeded` → clear `payment_failing` flag
- `customer.subscription.deleted` → downgrade to FREE

  }

## Rate Limits

| Tier | Rows/day | Reset |
|------|----------|-------|
| FREE | 500 | Daily (UTC midnight via Redis TTL) |
| PAID ($9/month) | 500,000 | Daily |

Row count = number of rows in the uploaded file.
A `/preview` call never deducts rows.
An `/explain` call deducts 1 row.

---

## Supported File Formats

| Format | MIME type | Notes |
|--------|-----------|-------|
| CSV | `text/csv` | UTF-8, comma-separated |
| JSON | `application/json` | Array of objects |
| xlsx | `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` | First sheet only; formulas → empty |

Max file size: **10 MB**. All formats return **CSV output**.
